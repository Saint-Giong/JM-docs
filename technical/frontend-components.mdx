## Overview

The Job Manager frontend is built using a **Headless UI Architecture** with Next.js, separating business logic (hooks) from visual presentation (UI components). This page provides detailed C4 component diagrams for each feature module.

<Info>
  All feature modules share common infrastructure: **Fetcher Utilities** for API calls, **Constants Manager** for URLs, and the **bamboo-ui** shared component library.
</Info>

## Architecture Pattern

<CardGroup cols={2}>
  <Card title="Headless Hooks" icon="code">
    Encapsulate business logic, state management, and event handlers without rendering UI
  </Card>
  <Card title="UI Components" icon="palette">
    Reusable visual primitives from bamboo-ui that consume hooks for behavior
  </Card>
  <Card title="Feature Services" icon="gear">
    API integration layer using Fetcher Utilities for backend communication
  </Card>
  <Card title="Validation Schemas" icon="shield-check">
    Zod schemas for client-side validation of form inputs
  </Card>
</CardGroup>

## Common Infrastructure

All feature components connect to these shared containers:

| Container | Technology | Purpose |
|-----------|------------|---------|
| **Page** | Next.js | Root page container and routing |
| **Fetcher Utilities** | TypeScript | HTTP/HTTPS requests to backend |
| **Constants Manager** | TypeScript | Centralized URL and config management |
| **bamboo-ui** | React + TailwindCSS | Shared UI component library |

## Applicant Search Component

Powers the "Headhunting" feature with advanced filtering and search capabilities.

<Frame>
  <img src="/images/diagrams/frontend/applicantsearch.png" alt="Applicant Search Component Diagram" />
</Frame>

### Components

<Tabs>
  <Tab title="Search Components">
    | Component | Purpose |
    |-----------|---------|
    | **Search Orchestrator Service** | Centralizes search logic, aggregates query parameters |
    | **SearchBar** | Renders search input with debounced handling |
    | **Skill Service** | Fetches available skills for autocomplete suggestions |
    | **SkillsFilter** | Renders skill tags input for filtering by tech stack |
  </Tab>
  <Tab title="Reusable Components">
    | Component | Hook | Purpose |
    |-----------|------|---------|
    | **Headless TagInput** | Headless TagInput Hook | Multi-select skill filtering |
    | **Headless SearchBar** | Headless SearchBar Hook | Debouncing and search state |
    | **Headless Input** | Headless Input Hook | Standard text filter fields |
  </Tab>
</Tabs>

### Search Flow

<Steps>
  <Step title="User Input">
    SearchBar captures keyword, SkillsFilter captures skills
  </Step>
  <Step title="Debounce">
    Headless SearchBar Hook delays API calls until typing stops
  </Step>
  <Step title="Orchestrate">
    Search Orchestrator Service aggregates all filter criteria
  </Step>
  <Step title="Execute">
    Fetcher Utilities sends search request to backend
  </Step>
</Steps>
